Errata workflow

Errata are introduced and stored in the issue list of a github repository. The workflow is as follows:

  • An issue is raised. The issue SHOULD also bear a label defined by for the repository (typically a reference to a specific document). (A specific label may also be defined by the repo owners to differentiate the proposed erratum from the other issues.)
  • The community discusses the issue. If it is accepted as a genuine erratum, the label Errata is added to the entry. Additionally, a new comment on the issue MAY be added, beginning with the word "Summary".
  • If the community rejects the issue as an erratum, the issue is closed.

The report retrieves all open issues with the label Errata, and displays the sections below. Each section collects the issues whose set of labels include a specific label defined for that section. Finally, a separate section MAY be added to display errata that do not fall in any of the label specific sections.
