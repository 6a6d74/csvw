Errata workflow

Errata are introduced and stored in the issue list of the group‘s GitHub repository. The workflow is as follows:

  • An issue is raised for a possible erratum. The label of the issue SHOULD be set to ErratumRaised. It SHOULD also include the label Model Document, Metadata vocabulary document, CSV to RDF mapping, or CSV to JSON mapping (adding other labels beyond one of those, or several of those, is o.k.).
  • The community discusses the issue. If it is accepted as a genuine erratum, the label Errata is added to the entry (and the ErratumRaised label is removed). Additionally, a new comment on the issue MAY be added, beginning with the word "Summary:" (if this is useful based on the discussion.)
  • If the community rejects the issue as an erratum, the issue is closed.

This report contains a reference to all open issues with the label Errata, displayed in the sections below. Each section collects the issues for a specific document.

If you have problems following this process, but you want nevertheless report an error, you can also contact the staff contact of the Working Group, Ivan Herman.
