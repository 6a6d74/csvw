CSVW Test Vocabulary

Test case manifest vocabulary extensions

Test Case Classes

This vocabulary extends the standard RDF Test Vocabulary with CSVW-specific tests.

csvwt:CsvSparqlTest
CSV SPARQL Test

A CsvSparqlTest takes an input file (mf:accept) in text/csv format to produce a an RDF graph which used as the default graph of a SPARQL ASK query contained in mf:result. The test entry MAY use :implicit files, have user-specified :metadata, and have a :httpLink header defining a describedBy relationship. Test Processors SHOULD act as if the file referenced via mf:action is returned using such a link header if it is not included in the HTTP response.

subClassOf: csvwt:PositiveEvaluationTest

csvwt:CsvValidationTest
CSV Validation Test

A CsvValidationTest takes an input file(mf:accept) in text/csv format to produce a JSON representation of the annotated tabular data model as defined in [[csvw-test-suite]]. The test entry MAY use :implicit files. The expected results are compared using JSON object comparison with the processor output.

subClassOf: csvwt:PositiveEvaluationTest

csvwt:CsvToJsonTest
CSV to JSON Test

A CsvToJsonTest takes an input file (mf:accept) in text/csv format to produce a an JSON file. The test entry MAY use :implicit files, have user-specified :metadata, and have a :httpLink header defining a describedBy relationship. Test Processors SHOULD act as if the file referenced via mf:action is returned using such a link header if it is not included in the HTTP response. The expected results are compared using JSON object comparison with the processor output.

subClassOf: csvwt:PositiveEvaluationTest

csvwt:CsvToRdfTest
CSV to RDF Test

A CsvToRdfTest takes an input file (mf:accept) in text/csv format to produce a an RDF graph which is isomorphic to the graph generated from the result file (mf:result) (see [[RDF-CONCEPTS]]). The test entry MAY use :implicit files, have user-specified :metadata, and have a :httpLink header defining a describedBy relationship. Test Processors SHOULD act as if the file referenced via mf:action is returned using such a link header if it is not included in the HTTP response.

subClassOf: csvwt:PositiveEvaluationTest

csvwt:ManifestValidationTest
Manifest Validation Test

A ManifestValidationTest takes an input file(mf:accept) in application/csvm+json format to produce a JSON representation of the annotated tabular data model as defined in [[csvw-test-suite]]. The test entry MAY use :implicit files. The expected results are compared using JSON object comparison with the processor output.

subClassOf: csvwt:PositiveEvaluationTest

csvwt:MetadataSparqlTest
Metadata SPARQL Test

A MetadataSparqlTest takes an input file (mf:accept) in application/csvm+json format to produce a an RDF graph which used as the default graph of a SPARQL ASK query contained in mf:result. The test entry MAY use :implicit files, have user-specified :metadata, and have a :httpLink header defining a describedBy relationship. Test Processors SHOULD act as if the file referenced via mf:action is returned using such a link header if it is not included in the HTTP response.

subClassOf: csvwt:PositiveEvaluationTest

csvwt:Metadata2JsonTest
Metadata to JSON Test

A Metadata2JsonTest takes an input file (mf:accept) in text/csv format to produce a an JSON file. The test entry MAY use :implicit files. The expected results are compared using JSON object comparison with the processor output.

subClassOf: csvwt:PositiveEvaluationTest

csvwt:Metadata2RdfTest
Metadata to RDF Test

A Metadata2RdfTest takes an input file (mf:accept) in application/csvm+json format to produce a an RDF graph which is isomorphic to the graph generated from the result file (mf:result) (see [[RDF-CONCEPTS]]). The test entry MAY use :implicit files.

subClassOf: csvwt:PositiveEvaluationTest

csvwt:NegativeSyntaxTest
Negative Syntax Test

A type of test specifically for syntax testing. Syntax tests are not required to have an associated result, only an action. Negative syntax tests are tests of which the result should be a parser error.

subClassOf: csvwt:Test

csvwt:NegativeEvaluationTest
Positive Evaluation Test

A Negative Evaluation test is successful when the result of processing the input file specified as mf:action results in the error identified by the literal value of mf:result. The specifics of invoking test, including the interpretation of options (and other input files are specified through sub-class.

subClassOf: csvwt:Test

csvwt:PositiveEvaluationTest
Positive Evaluation Test

A Positive Evaluation test is successful when the result of processing the input file specified as mf:action exactly matches the output file specified as mf:result using the comparison defined in another class. The specifics of invoking test, including the interpretation of options and other input files are specified through a sub-class.

subClassOf: csvwt:Test

csvwt:PositiveSyntaxTest
Positive Syntax Test

A type of test specifically for syntax testing. Syntax tests are not required to have an associated result, only an action.

subClassOf: csvwt:Test

csvwt:Option
Processor Options

Options passed to the test runner to affect invocation of the appropriate API method.

csvwt:Test
Superclass of all CSVW tests

All CSVW tests have an input file referenced using mf:action. Positive and Negative Evaluation Tests also have a result file referenced using mf:result . Other tests may take different inputs and options as defined for each test class.

subClassOf: mf:ManifestEntry
