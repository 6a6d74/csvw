Detailed Use Case 1

Use cases will follow a narrative and, where possible, include examples of real data files.

Each use case will:

  • indicate the contributor.
  • (where necessary) reference a more detailed use case description - noting that this document should contain sufficient information to be able to stand alone.
  • reference specific requirement(s); e.g. "Requires: Requirement 1" - these references will be embedded in the narrative.
